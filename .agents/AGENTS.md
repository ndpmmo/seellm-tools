# SeeLLM Tools - Agent Guidelines

## Strict Rules
- **Versioning & Changelog**: Every single change, optimization, modification, or bug fix MUST be documented in detail in `CHANGELOG.md`.
- **Patch Version Bump**: The version in `package.json` must be bumped by +1 patch level for every release.
- **Git Flow**: All changes must be added (`git add`), committed with a clear prefix (e.g., `fix(...)`, `feat(...)`), and pushed (`git push`) to the remote repository.
